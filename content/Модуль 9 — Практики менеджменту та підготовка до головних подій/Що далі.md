## Завершення курсу: повна картина виробничого PPC-менеджменту

**Мета**: Звести воєдино 8 модулів у єдиний weekly workflow, провести самоперевірку, і дати орієнтири для подальшого розвитку.

---

## 1. Повний weekly workflow: Skill 1 → Skill 2 → Skill 3

Курс вчив не окремим «трюкам PPC», а виробничому циклу з трьох шарів. Ось як вони складаються у тижневий ритм.

### Skill 1 — Підготовка даних і target ACoS (Модуль 5 Крок 1)

**Коли:** початок тижня (понеділок-вівторок).
**Вхід:** Sellerboard Товари × 4 тижні, PPC Dashboard per portfolio, passport/context від менеджера.
**Вихід:**
- Таблиця target ACoS per ASIN (`max(Margin, BE-ACoS)`, cap 30, floor 10).
- 5-priority групування ASINs.
- Маркування `[EVENT]` / `[SEASONAL]` портфоліо для виключення з автоматичного bulk (Модуль 8 Крок 3–5).
- Reality checks: COG=0, Margin=NULL, ACoS format — блокери перед продовженням.

### Skill 2 — Тижневий bulk (Модулі 5, 7)

**Коли:** середина тижня (середа-четвер).
**Вхід:** bulk-файл з Seller Central + target ACoS таблиця з Skill 1.
**Дії (автоматичні правила):**
- Плейсменти ToS/RoS/PP по lookup-таблиці (Модуль 5 Крок 2).
- Бід reduction (No-click, Scavenger, chronic bleed) (Модуль 5 Крок 3–4).
- Audience / Business ±5% (Модуль 5 Крок 5).
- ST негативи 4-тижневою агрегацією (Модуль 5 Крок 6).
- Портфоліо-неймінг `[XX%] ASIN - name`.

**Вихід:** bulk, який можна одразу завантажити в Seller Central.

### Skill 3 — Manual Review (Модуль 6)

**Коли:** в парі з Skill 2, читати до або після завантаження.
**Вхід:** все те ж, що й Skill 2, плюс додаткові WoW/trend метрики.
**Вихід:**
- Текстовий звіт з підсумком і розбивкою по 43 тригерах, пріоритети 🔴🟡🟢.
- Bulk з варіантами (не можна заливати в Amazon без редагування).
- Chrome Extension промпти для listing-related fix.

### Додаткові шари (Модуль 8)

- **Competitor analysis** (Крок 1) — щотижневий огляд Helium 10 Keyword Tracker, Cerebro на конкурентів, реакція на зміни рангів.
- **Discount analysis** (Крок 2) — детектування Deals/Coupons конкурентів, рішення про counter-discount на основі margin economy.
- **Variation PPC** (Крок 3) — ревізія структури per variation, виведення у чіткі per-child campaigns.
- **Seasonality** (Крок 4) — передбачуваний ramp-up за 14–21 днів до піку.
- **Event preparation** (Крок 5) — Prime Day / Black Friday / Spring Sale: +15–20% біди, stock thresholds, hourly monitoring.

### Тижневий ритм (приклад)

```
Пн: Skill 1 — target ACoS, 5-priority групування, checks.
Вт: Competitor review (Keyword Tracker), seasonal calendar check.
Ср: Skill 2 bulk generation.
Чт: Skill 3 Manual Review, обробка 🔴🟡 тригерів, manager communication.
Пт: Bulk upload, post-upload verification; planning для наступного тижня.
```

У тижні перед event'ом — додатково daily checks за 3 дні до, hourly під час події.

## 2. Self-assessment по модулях

Замість пройти абстрактно — конкретні завдання, які треба вміти виконати самостійно на реальному акаунті.

### Модуль 1–2 (термінологія і метрики)

- [ ] Пояснити без конспекту: ACoS, TACoS, CR, CTR, BE-ACoS, Target ACoS, Impression Share.
- [ ] У Sellerboard знайти: Net Margin, COG, Net Profit per ASIN за тиждень.
- [ ] У Seller Central → Reports → Business: витягнути Sessions, Session Percentage, Unit Session Percentage.

### Модуль 3 (ключові слова і структура)

- [ ] Дослідити нішу: 3 core ключі → 20+ long-tail через Cerebro / AsinSight / Magnet.
- [ ] Спроектувати структуру: Auto, Broad, Exact per ASIN + harvest-flow з Broad у Exact.

### Модуль 4 (налаштування кампаній)

- [ ] Створити SP KW Exact з чистим budget / daily / bid на реальному ASIN.
- [ ] Виставити ToS/RoS/PP modifiers відповідно до Модуль 5 Крок 2 lookup.
- [ ] Запустити SB + SD без помилок.

### Модуль 5 (оптимізація — тижневий bulk)

- [ ] Порахувати target ACoS per ASIN для 10+ ASIN акаунту.
- [ ] Виконати плейсмент-оптимізацію через lookup-таблицю.
- [ ] Застосувати bid reduction правила: No-click, Scavenger, chronic bleed.
- [ ] Згенерувати ST негативи на 4-тижневій агрегації.
- [ ] Переіменувати портфоліо у `[XX%]` формат.

### Модуль 6 (Manual Review, 43 тригери)

- [ ] Розпізнати топ-10 тригерів: #1 (COG), #2 (WoW >50%), #7 (Sessions drop), #22 (Competition), #26 (CR drop), #43 (chronic bleed) і ще 4.
- [ ] Прочитати Підсумок звіту і вибрати, з чого починати.
- [ ] Відрізнити 🔴 від 🟢 за способом реакції.

### Модуль 7 (додаткові матеріали)

- [ ] Використовувати AI-інструменти (Grok, ChatGPT) для bid recommendation, ST класифікації.
- [ ] Helium 10 Cerebro / Keyword Tracker — робочий рівень, не «вивчав один раз».
- [ ] Data Dive RankRadar — перевірка рангів у ніші.

### Модуль 8 (менеджмент і події)

- [ ] Виконати competitor review на 1 ASIN за 15 хв (Keyword Tracker + Cerebro + Amazon Search).
- [ ] Детектувати Deal/Coupon конкурента і вибрати counter-strategy за margin decision tree.
- [ ] Ревізувати variation PPC structure: знайти канібалізацію ключа, реструктуризувати.
- [ ] Побудувати seasonal ramp-up plan за 21 день до піку.
- [ ] Підготувати event deal (Prime Day): discount, бід +15–20%, stock check, hourly monitoring plan.

Якщо хоч один чек-бокс нижче 70% впевненості — повернутися до відповідного модуля, повторно виконати практичні завдання і розбори кейсів.

## 3. Ресурси для подальшого розвитку

### Щоденно / щотижнево

- **Seller Central News** — Amazon регулярно змінює rules, fees, event dates.
- **Helium 10 / Cerebro** — щотижневе читання rank-рухів.
- **Sellerboard** — margin watching (Net Margin, Refund Rate, COG warnings).
- **Keepa** — ціна конкурентів, BSR рухи.

### Навички вище базового

- Облік Sponsored Brands Video і SD Audience (Модуль 4 не вистачає глибини по цих форматах — опанувати через Amazon Ads Help + Helium 10 вебінари).
- Attribution / Multi-touch — складна тема, але стає важливою для акаунтів >$100k/міс.
- Amazon DSP — для брендових акаунтів з окремим бюджетом.

### AI-інструменти

- **Grok / ChatGPT** — швидке прототипування ST класифікації, bid recommendation логіки. Не замінює Skill 2 / Skill 3, але прискорює debug.
- **Claude (цей курс) / GPT-4** — довгий контекст, добре для code review bulk-generators, розбору тригерів.

### Команда і комунікація

PPC-manager **не працює ізольовано**. Залежності:
- **Account manager** — discount rules, business priorities, stock.
- **Listing manager** — content, images, A+.
- **Ops / supply chain** — FBA shipment timing.

Weekly sync з account manager — обов'язковий ритуал. Результати Manual Review показувати з контекстом: не «тригер #43 на кампанії X», а «кампанія X палить $60/тиждень без orders 3 тижні поспіль, пропоную варіанти A/B/C».

## 4. Фінальна перевірка

**Ти готовий вести PPC акаунт самостійно, якщо:**

1. За 2–3 години можеш повторити повний тижневий цикл (Skill 1 → 2 → 3) на 1 акаунті з 15+ ASIN.
2. Розрізняєш, коли `ACoS > target` — це баг у налаштуваннях, а коли — нормальний event-ефект.
3. Умієш ескалювати: формулюєш питання account manager'у з **варіантами дій** (не «що робити?», а «пропоную A з ризиком X, B з ризиком Y — обирай»).
4. Не лінуєшся писати optimization log після кожної суттєвої зміни — це твій інструмент проти забування.
5. Приймаєш, що 20% рішень будуть помилковими. Фіксуєш їх, вчишся, коригуєш. Це **виробництво**, а не іспит.
