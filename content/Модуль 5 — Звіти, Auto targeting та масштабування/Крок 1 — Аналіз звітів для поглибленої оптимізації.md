**Мета**: Навчитися витягати з Search Term Report, Placement Report і Purchase Product Report не поверхневі висновки («ACoS великий — понижай»), а рішення: що негативити, що виносити, що розривати у варіації. Три звіти — три різні логіки агрегації, різні пороги.

---

## 1. Search Term Report — 4-тижнева агрегація, а не тижневий снапшот

**Ключова теза:** тижневий ST-снапшот ненадійний. Amazon-аукціон шумний — погана погода в аукціонах тижня дає ілюзію «поганого» ST, який наступного тижня конвертить нормально. Тому негативимо **тільки по 4 тижнях поспіль**.

### Логіка агрегації

1. Завантажити SP Search Term Report за 4 окремі тижневі bulk (не 28 днів одним файлом — Amazon агрегує інакше).
2. Сумувати per-search-term: Spend, Clicks, Orders.
3. Якщо <4 тижневих файлів → ST йде у **manual_candidates** (не додаємо автоматично).

### Тригери для negativeExact

| Умова за 4 тижні | Дія |
|-------|-----|
| Spend ≥ $10, 0 ордерів | **negativeExact** |
| 15+ кліків, 0 ордерів | **negativeExact** |
| ACoS < target ACoS | **НЕ негативити** (профітний з шумом) |

> [!warning] У Exact-кампаніях ніколи не негативимо ST.
> Exact вже звужений до точних збігів — негатив там б'є по основному таргету. Негативи додаємо тільки в Broad / Phrase / Auto.

### Нормалізація перед додаванням

- lowercase усі терми
- strip articles/prepositions (a, an, the, for, of, in, on...)
- об'єднати словоформи (knives / knive → knife)
- порівняти з наявними `Campaign Negative Keyword` і `Negative Keyword` — якщо вже є, пропускаємо

Без нормалізації Amazon запише «best kitchen knives», «Best kitchen knife», «best the kitchen knives» як три окремі негативи.

Деталі логіки і приклади — у **Модуль 5, Крок 5, розділ 3**.

## 2. Placement Report — коротко

> [!note] Повна логіка Placement оптимізації (Top of Search / Rest of Search / Product Pages модифікатори, placement compensation) — **Модуль 5, Кроки 2-3**.

Тут фіксуємо тільки **що читати** зі звіту:

- **Колонки**: Placement, Impressions, Clicks, Spend, Sales, ACoS, CTR, CR.
- **Сортувати**: за ACoS (де ACoS > target у 1.5×, дивимося на кліки — чи є статистика для рішення).
- **Мінімум для висновку**: 7+ кліків на плейсмент. Менше — шум.
- **ToS impressions <20% від загальних** — сигнал, що база-бід замалий або ToS modifier негативний. Це окремий тригер #36 у Модулі 6.

Звичайна оптимізація плейсментів виконується **через bulk-файл**, а не через UI — щотижня, без звітів. Звіт використовуємо для авдиту structure: чи розумно у ASINа розподіляється трафік по плейсментах.

## 3. Purchase Product Report — маржинальна математика

**Ключова теза:** Purchase Product показує, що саме купили по нашій рекламі. У варіаціях часто **рекламований ASIN ≠ куплений ASIN**. Якщо куплений має нижчу маржу, реклама з'їдає профіт, хоча в звіті виглядає ОК.

### Як рахувати

Для кожного Purchased ASIN ≠ Advertised ASIN:

```
Unit Profit (куплений) = Price − FBA fees − Referral fee − COG
Effective ACoS         = Spend / Sales × 100%
Real margin per order  = Unit Profit − (Spend / Orders)
```

### Приклад (конкретні числа)

| Показник | Advertised (ASIN A, 9-slot rack) | Purchased (ASIN B, 3-slot rack) |
|----------|----------------------------------|----------------------------------|
| Price | $29.99 | $12.99 |
| FBA + Referral | $6.50 | $3.80 |
| COG | $8.00 | $5.20 |
| Unit Profit | **$15.49** | **$3.99** |
| Orders (за тиждень) | 5 | 3 |
| Sales | $149.95 | $38.97 |
| Spend attribution | $35 | $15 |
| CPO (Spend / Orders) | $7 | $5 |
| Real profit per order | $15.49 − $7 = **$8.49** | $3.99 − $5 = **−$1.01** |

ASIN B горить: кожен ордер мінус долар. Дія — див. розділ 4 і **Крок 4** цього модуля.

### Дії залежно від результату

- **Real profit > 0** і purchased має стабільний обсяг → створити окрему кампанію для нього (Exact keyword або Product Targeting).
- **Real profit < 0** → розривати варіацію або ставити Negative Product Target на low-margin ASIN у батьківській кампанії (якщо варіація не розривається — ескалація менеджеру).
- **Різниця маржі >3×** і обсяги значущі → **завжди** ескалація, бо рішення «розривати варіацію» — це не самостійна дія менеджера.

---

## 4. Коли використовувати звіти, а коли — bulk / UI

| Ситуація | Інструмент |
|----------|------------|
| Щотижнева рутинна оптимізація одного акаунту | **Bulk-файл** (Модуль 5) |
| Авдит структури акаунту (1 раз на квартал) | Усі 3 звіти + Sellerboard |
| Пошук harvest-кандидатів | ST Report + тригер #40 (Модуль 6) |
| Перевірка, чи не «їдять» варіації одна одну | **Purchase Product Report** обов'язково |
| Оптимізація плейсментів у стабільній кампанії | Bulk, не звіт |

Репорти — для **масових перевірок** і побудови зведених таблиць. Для щоденних рішень — bulk-файл через скрипт.

### Практичний кейс

Purchase Product Report для ASIN A (chef knife, Advertised). Дані за 4 тижні:

| Purchased ASIN | Orders | Sales | Spend | Unit Profit | CPO |
|----------------|--------|-------|-------|-------------|-----|
| A (сам) | 12 | $360 | $48 | $18 | $4 |
| B (paring knife, варіація) | 5 | $75 | $35 | $4 | $7 |
| C (steak knife set, варіація) | 2 | $90 | $20 | $22 | $10 |

Target ACoS = 20%, ASIN A margin = 40%.

**Завдання:**
1. Real profit per order для A, B, C?
2. Який ASIN горить, які прибуткові?
3. Які дії для B і C?

*Розв'язок:*
1. Real profit per order:
   - A: $18 − $4 = **+$14**
   - B: $4 − $7 = **−$3**
   - C: $22 − $10 = **+$12**
2. B горить (−$3/ордер), A і C прибуткові. Хоча Sales у B вищі за Spend ($75 vs $35), маржа з'їдається CPO.
3. Дії:
   - **B**: Negative Product Target по ASIN B у кампанії на A. Якщо обсяг значущий — ескалація менеджеру для оцінки розриву варіації.
   - **C**: створити окрему кампанію Sponsored Products (Product Targeting або Keyword Exact) під C — він рентабельніший за A у перерахунку на ордер.

### Розрахункова задача

ST «premium kitchen knife» у кампанії `SP Kw Br knife-set`. Дані за 4 тижні:

| Тиждень | Spend | Clicks | Orders |
|---------|-------|--------|--------|
| W1 | $3.20 | 4 | 0 |
| W2 | $4.10 | 5 | 0 |
| W3 | $2.80 | 4 | 0 |
| W4 | $3.50 | 5 | 0 |

Target ACoS = 22%.

**Порахуйте:**
1. Сумарний Spend за 4 тижні?
2. Сумарна кількість кліків?
3. Які тригери спрацьовують?
4. Дія?

*Розв'язок:*
1. Spend = 3.20 + 4.10 + 2.80 + 3.50 = **$13.60**
2. Clicks = 4 + 5 + 4 + 5 = **18**
3. Тригери:
   - Spend ≥ $10 за 4 тижні, 0 ордерів ✅
   - 15+ кліків за 4 тижні, 0 ордерів ✅
4. Дія: **negativeExact у `SP Kw Br knife-set`**. Нормалізувати: lowercase «premium kitchen knife», перевірити, чи нема вже у `Negative Keyword` / `Campaign Negative Keyword`.
