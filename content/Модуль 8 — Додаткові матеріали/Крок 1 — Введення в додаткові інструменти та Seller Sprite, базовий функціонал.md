**Мета**: Зрозуміти, яке місце займають додаткові інструменти у виробничому процесі PPC-менеджера, і освоїти базовий функціонал Seller Sprite для аналізу SERP — щоб у потрібний момент Skill 1 / 2 / 3 ти міг за 60 секунд пояснити «чому ASIN просів» або «хто тримає ToS».

> [!warning] Інструменти — не самоціль. У роботі з 30-акаунтним портфоліо у тебе НЕМАЄ часу «походити по Seller Sprite подивитись». Кожен інструмент вмикається в конкретній точці workflow, відповідаючи на конкретне питання. Цей крок вчить саме цьому: коли і навіщо.

## 1. Екосистема інструментів: де який підключається

Виробничий PPC-процес спирається на три скіли:

- **Skill 1 — Data collection & target ACoS** (Sellerboard Товари + PPC Dashboard по портфоліо).
- **Skill 2 — Bulk optimization** (правила по плейсментах, ключах, бюджету).
- **Skill 3 — Тригери/діагностика** (коли щось просіло — знайти причину).

Додаткові інструменти розподіляються так:

| Інструмент | Основна роль | Де підключається |
|------------|--------------|------------------|
| **Seller Sprite** | SERP-фото + листинг-diff (ціна / BSR / хайджеки / зміни картинок) | Skill 3 тригер #7 (Sessions drop), #32 (CTR drop), Skill 1 огляд конкурентів |
| **Sellerboard** | Margin, COG, Real ACoS, Sessions, USP, Break-even ACoS | Skill 1 (source of truth для target ACoS), Module 5 Step 1 |
| **AsinSight** | Трафік ASIN + keyword research | Module 8 Step 1 (competitor analysis), Module 6 тригер #22 |
| **Data Dive** | Niche + RankRadar (органічні позиції) | Module 6 ranking reminder, Module 4 (keyword research для нових кампаній) |

**Ключова думка:** якщо твій звіт по тижневій оптимізації не посилається на жоден із цих інструментів — це тривожний знак. Або ти пропустив діагностику, або «оптимізував» без перевірки зовнішніх факторів.

## 2. Seller Sprite: що це і чому ставимо першим

Seller Sprite — це безкоштовне розширення для Chrome. Робить дві речі, які рятують час:

1. **SERP-layer** — коли ти вводиш ключ у Amazon, під кожним ASIN з'являється табличка (бренд, ціна, BSR, продавці, SP/Organic-позиція).
2. **Listing-overlay** — коли ти відкриваєш картку товару, показує графіки змін (ціна / BSR / рейтинг / кількість продавців / хайджеки) за 7 / 30 / 90 днів.

**Встановлення:**
1. Зайди на [sellersprite.com](https://www.sellersprite.com/).
2. Постав розширення з [Chrome Web Store](https://chromewebstore.google.com/detail/sellersprite-amazon-resea/lnbmbgocenenhhhdojdielgnmeflbnfb).
3. Авторизуйся через email — базовий функціонал безкоштовний.

**Чому саме Seller Sprite, а не Helium 10 / Jungle Scout:** безкоштовна версія Seller Sprite дає ті 80% даних, які реально потрібні PPC-менеджеру для SERP-check і «що там у конкурента за 30 днів». Для keyword-research ми все одно йдемо в AsinSight / Data Dive (Крок 4 / 5).

## 3. SERP-аналіз: як працює і коли використовувати

**Що показує:**
- SP Position (рекламна позиція в Sponsored Products — наприклад, SP #1 означає топ рекламного блоку).
- Organic Position (органічна позиція).
- Бренд, ціна, BSR, кількість продавців на картці.

**Процес:**
1. Відкрий Amazon, введи ключ (наприклад, `ice cream scoop`).
2. Увімкни розширення (іконка в Chrome).
3. Під кожним ASIN у видачі з'явиться таблиця з даними.
4. Порівняй свій ASIN з топ-5 (SP + Organic).

### Коли використовувати (привʼязка до workflow)

**Трігер Skill 3 #7 — Sessions drop.** Коли Sellerboard показує, що Sessions впали >20% WoW по топ-ASIN, перше запитання: «А що з видачею?». Відкриваєш SERP Seller Sprite за головним ключем цього ASIN — якщо твоя позиція випала з топ-10 SP, це одразу пояснює дроп (імпресії впали → сесії впали).

**Трігер Skill 3 #32 — CTR drop.** CTR падає зазвичай через (а) погіршення картинки, (б) зміни цін конкурентів, (в) втрату якісної позиції. SERP-фото у Seller Sprite дає перші два сигнали: порівняй ціну свого ASIN з топ-3, подивись чи у них з'явилися купони / нові картинки.

**Module 6 тригер #22 — competition pressure.** Якщо конкуренти раптом «ущільнилися» на ToS (3+ SP-позицій від різних брендів за твоїм головним ключем за останній тиждень), це точка входу для тригера #22 — перевірка на structural change у ніші.

**Skill 1 — competitor watch (раз на тиждень).** Під час збору target ACoS раз на тиждень роби SERP-check по топ-5 ASIN по спенду. 10 хвилин часу, можна помітити зміни раніше, ніж вони проявляться у Spend / ACoS.

## 4. Що робити з даними SERP

Три можливі висновки і їхні наслідки для PPC:

**Висновок A: «Твоя позиція хороша (топ-5), конкуренти не змінились»** → нічого не роби, SERP не пояснює проблему, шукай далі (Sessions, CR, ціна, лістинг).

**Висновок B: «Твоя позиція випала з топ-10, конкурент на ToS з нижчою ціною»** → структурна проблема:
- Перевірити свої ставки по головному ключу (чи не різко знижені).
- Перевірити стан лістингу (чи не з'явився suppressed bullet / інша проблема).
- Запропонувати аккаунт-менеджеру перевірити ціну.

**Висновок C: «SERP без змін, але CTR все одно впав»** → проблема у картці (нова негативна фотка? відгук 1 зірка?). Переходь до Кроку 2 Seller Sprite (listing-overlay).

---

## 5. Обмеження Seller Sprite

- **Реклама не завжди точна.** Seller Sprite показує SP-позицію тільки якщо конкретно в цей момент туди попало оголошення. Для повної картини ранків — AsinSight / Data Dive RankRadar.
- **Купони не відображаються.** Треба перевіряти вручну на картці.
- **Частина даних за 90 днів — тільки у платній.** Для PPC-менеджера безкоштовної версії достатньо на 90% випадків.

### Практичний кейс

**Сценарій.** Тобі приходить ASIN `B07AAA` у Групу 1 (Skill 1 сортування) через просадку Net Profit −22% WoW. Ти відкриваєш PPC Dashboard — ACoS підріс з 19% до 27%, Spend виріс на 15%. Sellerboard показує Sessions −18% WoW, CR −0.8 pp. Інше по акаунту стабільно.

**Питання:** з якого інструменту почати діагностику і чому? Який конкретно екран/функція? Що чекаєш побачити?

*Розв'язок:*

Починати — з **Seller Sprite SERP-check** (не з AsinSight, не з Sellerboard, не з Data Dive). Обґрунтування:
- Sessions впали → проблема з видимістю. Перший крок — переконатись що ASIN не «злетів» з топ-10.
- Seller Sprite дає відповідь за 60 секунд, AsinSight / Data Dive — за 10–15 хв.
- Це дешевий у часі фільтр: якщо SERP OK — переходимо до listing-overlay (Крок 2), потім до AsinSight (Крок 4) якщо перший не дав відповіді.

Що очікуєш побачити: або (а) свій ASIN випав з топ-5 SP → треба перевірити біди на головному ключі і проконсультуватись з аккаунт-менеджером про ціну; або (б) конкурент зайшов з ціною нижче твоєї → проблема ціни, PPC не вилікує.

### Розрахункова задача

За даними Seller Sprite SERP по ключу `ice cream scoop`:

| ASIN | SP pos | Organic pos | Ціна | BSR |
|------|--------|-------------|------|-----|
| Твій B07XYZ123 | — | 12 | $12.99 | 850 |
| Конкурент B08ABC456 | 1 | 3 | $9.99 | 210 |
| Конкурент B08DEF789 | 2 | 5 | $11.49 | 430 |

Твоя кампанія SP Kw Ex на цей ключ: ACoS 34%, Margin ASIN 18%, BE-ACoS 15%.

**Порахуйте:**
1. Optimization target ACoS.
2. Наскільки фактичний ACoS перевищує target (у pp і у разах).
3. Яка ймовірна причина — вища ціна чи слабка позиція? Який конкретний аргумент підкажуть дані?

*Розв'язок:*
1. target = max(Margin 18, BE-ACoS 15) = 18%. Підтягування до floor 10% не потрібне, cap 30% не спрацьовує.
2. 34% − 18% = **16 pp перевищення**, 34 / 18 ≈ **1.89× target**.
3. Обидва фактори діють. Аргументи:
   - **Ціна**: твоя $12.99 vs $9.99 у конкурента з SP #1 → різниця 30%, це прямо б'є по CR.
   - **Позиція**: Organic #12 поза топ-10 → органічних сесій суттєво менше, і ти сплачуєш PPC за трафік, який органічно вже не йде.
   → Рекомендація: не підвищувати біди далі (ACoS уже у 1.89× target), а йти до аккаунт-менеджера з пропозицією перегляду ціни / купона і паралельно перевірити чи є місце для листинг-змін (Крок 2).
