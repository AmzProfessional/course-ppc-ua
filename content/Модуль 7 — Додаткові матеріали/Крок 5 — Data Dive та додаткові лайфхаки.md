> [!note] **Примітка**: Перегляньте відео (37:08–59:58), яке пояснює використання Data Dive (Nishi, RankRadar) для аналізу ніш і позицій, а також лайфхаки для AsinSight (аналіз варіацій, Temp Mail) і Sellerboard (аналіз профіту, утримань).

## 1. Значення Data Dive та лайфхаків

> [!note] Data Dive — платний інструмент для аналізу ніш і відстеження органічних позицій, який доповнює PPC-стратегію, а лайфхаки оптимізують роботу з AsinSight і Sellerboard.

![[Pasted image 20250903213401.png]]

**Чому це важливо?**

- **Niche**: Дозволяє аналізувати конкурентів і ключі в ніші, визначати релевантність ключів.
- **RankRadar**: Відстежує органічні позиції ASIN за ключами, допомагає виявляти просідання чи ріст.
- **Лайфхаки**:
    - AsinSight: Аналіз варіацій і безкоштовний доступ через Temp Mail.
    - Sellerboard: Точний аналіз профіту, утримань і продажів (SP, SB).
- **Оптимізація PPC**: Дані з Data Dive допомагають вибирати ключі та коригувати ставки.

**Приклад**: RankRadar показує просідання ASIN із 3-ї до 10-ї позиції за ключем "spice jars".

## 2. Аналіз через Data Dive

### 2.1. Niche

- **Як працює**:
    1. Увійдіть у Data Dive (доступи надає керівник).
    2. Виберіть нішу (Niche), додану аккаунт-менеджером за ключем або ASIN.
    3. Перегляньте таблиці: ASIN, конкуренти, ключі, продажі, ціни, рейтинги.
    4. Сортуйте ключі за Search Volume (SV), релевантністю чи позиціями.
       
       ![[Pasted image 20250903213651.png]]
       
- **Що аналізувати**:
    - **Конкуренти**: ASIN із Strong/Very Strong статусом (високі продажі, відгуки).
    - **Ключі**: Релевантні ключі з SV >5,000 для PPC.
    - **Вкладки**:
        - **Master Keyword List (MKL)**: Основні ключі ніші.
          
          ![[Pasted image 20250903213701.png]]
          
        - **Outliers**: Широкі ключі з високим SV, але низькою релевантністю.
        - **Roots**: Кореневі слова для broad/phrase кампаній (наприклад, "spice" для "spice jars").
        - **Normalizer**: Уникайте дублювання ключів (наприклад, "garlic press kitchen" = "garlic press for kitchens").
- **Приклад**: Ніша "pet treat molds", ваш ASIN на 2-й позиції, ключ "pet treat mold" (SV 8,000). Рішення: додати до exact match кампанії.

### 2.2. RankRadar

- **Як працює**:
    1. У Data Dive → RankRadar → Додайте ASIN і ключі (узгодьте ліміти з менеджером).
    2. Перегляньте таблиці: органічні позиції за ключами, зміни за 7–30 днів.
    3. Сортуйте за SV, релевантністю чи позиціями.
       
       ![[Pasted image 20250903213924.png]]
       
- **Що аналізувати**:
    - **Позиції**: Просідання (наприклад, із 3-ї до 10-ї) чи ріст (із 10-ї до 3-ї).
    - **Ключі**: Релевантні ключі з SV >5,000 для посилення ToS.
    - **Варіації**: Дані включають усі варіації ASIN, перевірте вручну в Amazon Search.
      
      ![[Pasted image 20250903214442.png]]
      
- **Приклад**: ASIN (B07XYZ123) за ключем "spice jars" упав із 3-ї до 10-ї позиції. Рішення: підняти ставку на ToS (+20%) і перевірити конкурентів.

### Чек-лист:

- [ ]  Відкрити Data Dive, вибрати нішу (Niche) за ключем або ASIN.
- [ ]  Проаналізувати конкурентів: топ-3 ASIN, їхні продажі, ключі (SV >5,000).
- [ ]  У RankRadar перевірити позиції ASIN за 7–30 днів, виявити просідання.
- [ ]  Додати 2–3 релевантні ключі до PPC (exact match, bid 0.50–0.75).
- [ ]  Перевірити вручну позиції в Amazon Search для варіацій.

## 3. Додаткові лайфхаки

### 3.1. Аналіз варіацій у AsinSight

- **Як працює**:
    1. У AsinSight → Asin Research → Введіть parent ASIN із варіаціями.
    2. Натисніть на parent item → Перегляньте розподіл трафіку за варіаціями (colors).
    3. Аналізуйте ключі та показники для кожної варіації.
- **Що робити**:
    - Визначте, яка варіація отримує більше трафіку (наприклад, 70% на parent ASIN).
    - Якщо варіація слабка (<10% трафіку), додайте PPC на її ключі.
- **Приклад**: Parent ASIN (70% трафіку), зелена варіація (28%), синя (<2%). Рішення: запустити PPC для синьої варіації на ключ "blue iPhone case".

### 3.2. Temp Mail для AsinSight

- **Як працює**:
    1. Перейдіть на Temp Mail (зелений конверт).
    2. Сгенеруйте тимчасову email і використайте її для реєстрації в AsinSight.
    3. Підтвердьте через лист із Temp Mail → Отримайте 30 днів безкоштовного доступу.
    4. Повторюйте щомісяця з новою email.
- **Чому**: Економить кошти на платну підписку AsinSight.

### 3.3. Аналіз профітуту та утримань у Sellerboard

- **Як аналізувати профіт**:
    1. У Sellerboard → Товари → Закази → Фільтруйте за ASIN.
    2. Перегляньте профіт за одиницю (переклад, без знижок/купонів).
    3. Порівняйте з маржинальністю PPC-кампаній.
- **Як аналізувати утримання**:
    1. У Dashboard → Утримання → Відніміть FBA Fee і Referral Fee від загальних утримань.
    2. Оцініть додаткові витрати (повернення, зберігання, доставки).
    3. Якщо утримання впливають на профіт, повідомте менеджера.
- **Лайфхак**: Для швидкості використовуйте таблиці з маржинальністю від аккаунт-менеджерів.
- **Приклад**: Утримання 20,000 доларів за тиждень, FBA+Referral 15,000 доларів → Additional - 5,000 доларів (повернення). Рішення: перевірити якість доставки.

### Чек-лист:

- [ ]  У AsinSight проаналізувати варіації ASIN, визначити слабкі (<10% трафіку).
- [ ]  Зареєструвати AsinSight через Temp Mail для безкоштовного доступу.
- [ ]  У Sellerboard перевірити профіт за одиницею і утримання (мінус FBA+Referral).
- [ ]  Повідомити менеджера про високі утримання чи слабкі варіації.
- [ ]  Зберегти таблицю з маржинальністю для PPC.

## 4. Ключові поради для PPC Manager’ів

- **Фокус на релевантність**: У Niche вибирайте ключі з високою релевантністю для PPC.
- **Відстежуйте позиції**: RankRadar допомагає швидко виявити просідання.
- **Перевіряйте варіації**: У AsinSight аналізуйте слабкі варіації для PPC.
- **Економте з Temp Mail**: Безкоштовний доступ до AsinSight щомісяця.
- **Аналізуйте утримання**: У Sellerboard віднімайте FBA+Referral для оцінки додаткових витрат.
